# 新用户快速上手：OpenClaw ⇄ OpenCode RouteA（ocbridge + TUI 插件）

> 适用对象：第一次使用本项目的同学（不需要了解三省六部流程）。
>
> 你将完成三件事：
> 1) 配好 NATS（你已有 VPS NATS 也可直接用）
> 2) 在 Worker 机器启动 `ocbridge`（常驻）
> 3) 在 OpenCode TUI 中用插件命令查看待办、切换模式、执行任务、回发消息

---

## 0. 你需要准备什么

### 必备
- 一台 **Worker 机器**（运行 OpenCode + ocbridge）
- 一台 **NATS 服务器**（推荐 VPS 上 `nats:2.10`，已启用 JWT/ACL）
- Worker 机已安装：
  - `python3`（建议 3.10+）
  - `git`
  - `opencode`（可执行：`opencode --version`）

### 推荐
- `nats` CLI（用于发/收测试消息）

---

## 1. 安装（Worker 机）

### 方式 A：一键安装（推荐）

```bash
curl -fsSL https://raw.githubusercontent.com/duoglas/opencode-openclaw-collab/main/scripts/install_routeA.sh | bash
```

脚本会交互式询问：`NATS_URL`、`NODE_ID`、`OCBRIDGE_MODE`、`NATS_CREDS` 路径，以及是否安装为 `systemd --user` 服务。

### 方式 B：手动安装

```bash
git clone https://github.com/duoglas/opencode-openclaw-collab.git
cd opencode-openclaw-collab
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

验证：
```bash
python -c "import ocbridge; print('ocbridge_ok')"
```

---

## 2. 配置 NATS（产品化建议：JWT + ACL）

### 2.1 NATS 地址
设置你的 NATS：
```bash
export NATS_URL='nats://<VPS_IP>:4222'
```

### 2.2 creds（强烈建议）
在 NATS 使用 JWT/ACL 时，Worker 需要一份 `.creds`：
- 建议文件名：`opencode_worker.creds`
- 放置路径：`~/.config/ocbridge/opencode_worker.creds`

并设置：
```bash
export NATS_CREDS="$HOME/.config/ocbridge/opencode_worker.creds"
chmod 600 "$NATS_CREDS"
```

> ACL 最小策略建议见：`docs/nats-acl-routeA.md`

---

## 3. 配置 ocbridge（Worker 机）

建议写一个 env 文件：`~/.config/ocbridge/env`

```bash
# NATS
export NATS_URL='nats://<VPS_IP>:4222'
export NATS_CREDS="$HOME/.config/ocbridge/opencode_worker.creds"

# 节点标识（影响 chat subject：oc.chat.to.<node_id>）
export NODE_ID='homepc-1'

# 执行能力
export CAPABILITY='coding'

# 执行模式：auto=无人值守自动执行；manual=有人值守先入队等待认领
export OCBRIDGE_MODE='manual'

# OpenClaw dispatch/result subjects（新+旧兼容）
export DISPATCH_SUBJECTS='openclaw.dispatch.v1,op.task.home'
export RESULT_SUBJECTS='openclaw.result.v1,op.result.controller'

# chat subject 约定（RouteA）
export CHAT_TO_PREFIX='oc.chat.to.'
export CHAT_FROM_PREFIX='oc.chat.from.'

# opencode serve
export OPENCODE_SERVE_HOST='127.0.0.1'
export OPENCODE_SERVE_PORT='4096'

# 本地落库
export OCBRIDGE_DB="$HOME/.local/share/ocbridge/bridge.db"
export OCBRIDGE_API_HOST='127.0.0.1'
export OCBRIDGE_API_PORT='7341'
```

> 注意：`node_id` 建议使用 `[a-z0-9_-]`，不要包含 `.`，避免 subject 被多段化。

---

## 4. 启动 ocbridge（Worker 机）

```bash
source ~/.config/ocbridge/env
cd ~/opencode-openclaw-collab
source .venv/bin/activate
python -m ocbridge.bridge_daemon
```

启动后 ocbridge 会：
- 确保本机 `opencode serve` 在 `127.0.0.1:4096` 可用
- 订阅 dispatch（如 `openclaw.dispatch.v1`）
- 订阅下行 chat：`oc.chat.to.<node_id>`
- 发布上行 chat：`oc.chat.from.<node_id>`（用于 `/oc-reply`）
- 本地用 SQLite 记录收发与执行轨迹

健康检查（可选）：
```bash
curl -s http://127.0.0.1:7341/status
```

## 5. 安装/启用 OpenCode 插件（TUI 里用命令交互）

本项目提供一个最小插件：`plugin/ocbridge`，用于在 TUI 里调用 ocbridge 本地 API。

> 由于 OpenCode 的插件加载方式可能因版本/配置不同而略有差异，这里给两种方式：

### 方式 A：把插件目录加入 OpenCode 配置（推荐）
1) 找到 OpenCode 配置路径：
```bash
opencode debug paths
```
2) 在 `~/.config/opencode/opencode.json` 或对应配置中，把本地插件路径加入（示例字段名可能是 `plugins`/`commands`，以你的版本为准）：
- 插件路径：`/home/<user>/opencode-openclaw-collab/plugin/ocbridge`

如果你不确定字段名：把 `opencode.json` 内容贴我，我按你的版本改。

### 方式 B：先不装插件，用 curl/本地 API 调试
你也可以先用：
```bash
curl -s http://127.0.0.1:7341/pending?limit=20
```
确认 ocbridge 正常工作后再接 TUI。

## 6. 在 TUI 里怎么用（最常用 6 个命令）

> 下面命令都在 OpenCode TUI 里输入（以 `/` 开头）。

### 6.1 模式切换
- 查看/切换执行模式：
```text
/oc-mode auto
/oc-mode manual
```

### 6.2 看队列与执行（manual 推荐流程）
```text
/oc-pending 20
/oc-claim <task_id>
/oc-run <task_id>
```

### 6.3 回发消息（chat 上行）
```text
/oc-reply <task_id> 我已开始执行，预计10分钟回传结果。
```

## 7. 最短端到端自测（建议你体验用）

### 7.1 控制端（任意机器）监听 worker 回包
```bash
nats sub "oc.chat.from.${NODE_ID}" --server "$NATS_URL" --creds <openclaw_dispatcher.creds>
```

### 7.2 控制端发一条下行 chat 给 worker
```bash
nats pub "oc.chat.to.${NODE_ID}" '{
  "schema":"oc.chat.v1",
  "task_id":"TASK-DEMO-CHAT-001",
  "from_id":"openclaw",
  "to_id":"node:'"$NODE_ID"'",
  "text":"请在TUI里回一句已收到，并执行一次最小任务",
  "ts": 0
}' --server "$NATS_URL" --creds <openclaw_dispatcher.creds>
```

### 7.3 Worker 在 TUI 执行
```text
/oc-inbox 10
/oc-reply TASK-DEMO-CHAT-001 已收到，开始处理。
```

你在 7.1 的订阅里应能看到回包。

## 8. 常见问题

### Q1：`/oc-reply` 报 permissions violation
说明 VPS NATS ACL 没放行 `oc.chat.from.*`（worker 上行）。请按：`docs/nats-acl-routeA.md` 调整。

### Q2：看不到 `oc.chat.to.<node>`
- 检查 `NODE_ID` 是否一致（控制端发的 node_id 和 worker 的 NODE_ID 要一样）
- 检查 worker 是否有权限 `sub oc.chat.to.*`（ACL）

### Q3：Worker 无人值守
把 `OCBRIDGE_MODE=auto` 并保证 worker 的 `opencode serve` 可用。
